\# Unity Development Journey

Unity 2022 — Built-in Render Pipeline



This repository documents my structured learning path in Unity through small, focused projects.

Each chapter represents a step in my progression as I experiment with movement systems, camera control, and gameplay mechanics.



\---



\## Structure



Each folder is a standalone Unity project:



* \### 01 - Car Controller

&#x20; \*\*Concepts practiced: Object movement, physics, collision detection, first-person camera, two-player input.

&#x20; \*\*Brief description: Car controller with obstacles to dodge, first-person camera mode and two-player support.



* \### 02 - Airplane Controller

&#x20; \*\*Concepts practiced: 3D movement, object rotation, animated component behavior.

&#x20; \*\*Brief description: Airplane controller with free flight and propeller rotation.



* \### 03 - Hunger

&#x20; \*\*Concepts practiced: Object spawning, state systems (hunger), projectiles, off-screen object cleanup.

&#x20; \*\*Brief description: Hunger mechanic prototype with randomly spawned animals and shooting system.



* \### 04 - Fetch

&#x20; \*\*Concepts practiced: Side and vertical spawning, lives system, scoring, game over, prefabs.

&#x20; \*\*Brief description: Collection game with spawned animals, player lives, scoring and game over screen.





\---



\## Purpose



This repository is not a polished game project.

It is a documented progression of experiments and implementations.



The goal is to:

\- Build core gameplay systems step by step.

\- Improve code clarity and structure over time.

\- Track my evolution as a Unity developer.
